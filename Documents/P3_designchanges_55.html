3. Design Changes

We have done some changes, after both feedback from our supervisor, and some things we found out in the process of coding.

Changes to frontpage.html: Initially we did not think about having a footer with contact information, but we got some inspiration from other websites and then we found out we wanted to have footer including this. We have a footer on all of our pages, and we think it makes the site more user-friendly and nice. We removed the social media icons we had on the frontpage and put it in the footer. We made an own JavaScript file including both the frontpage and the footer and linked it to the frontpage.html. We also included 5 pictures in the front page, instead of 3-4. We also cropped the images on the frontpage to 2200 x 840px. Other than that, the front page is as similar to how we imagined it would be.

Changes to openinghours.html: We centered the opening hours instead of having it on the same line as the map.

Changes to menu.html and information.html: On menu.html, we changed "Ikke-vegetar" to "Kjøttretter". In the information.html, we centered all of the text.

Changes to course.part and topbar.part: As we started the process with coding, we found out we did not need this. Instead we made different Javascripts and linked it to the html pages.

Changes to topbar.js, frontpage.js, courses.js, information.js: We decided to make three JavaScript files instead of four. We made one for the frontpage, one for the menu and then one for both the topbar and footer. We didn't need the last one called information.js, because there were no JavaScript needed (except topbar and footer) for that page.

Changes to style.css: We have one main css for the different pages, but we decided to make one external stylesheet for the topbar and footer. We thought this would make it a bit more organized and clean.

As for the CSS styling of the pages, we did some changes: We have changed all of the h1-elements from 95px to 70px, to fit both larger and smaller screens. We also changed the font size on "Informasjon" and "Åpningstider to 32px. The buttons on the menu site is scaled to width:60% and height: 70px, instead of width:100%. When hovering over the e-mail and phone links, we wanted it to be in the same color as the other links on the pages: #A0C693.

Additional

  • We have added topbarfooter.js, as mentioned above.
  • When discussing the top bar, we decided that when scaling the site (less than 1024px), the original top bar dissappears and a hamburger icon/button appears. The hamburger button includes the different alternatives on the top bar. Our logo will float to the left side of the page, while the rest of the top bar goes inside the hamburger button. Our logo is also a button for the frontpage.
  • We added some pictures for the front page called front_image1,2,3,4,5.png.
  • Furthermore, we added a avocado favicon on our page, called favicon.png.
  • We also added pictures of the logo on the topbar, and the social media buttons inside the footer.

Changes to JavaScript and modularity requirements

We had to change and add some JavaScript applications since the fourth JavaScript requirement, the email-feature, did not include JavaScript. We then decided to include the hamburger icon as described above, to make the site more user-friendly for all devices. We removed the requirement about the top bar being pinned to the top, and when the page scrolls down, its size changes. Instead we added the navigation bar and the footer to JavaScript for modularity.

Our JavaScript requirements is as mentioned below:

  1. The front page changes the background pictures every 5 seconds to highlight different pictures from the café and the navigation bar should always be visible at the top.
  2. The different elements on the menu become visible if the menu-button is clicked. Each element corresponds to a category of courses. When the user clicks on a category element, courses from this category will be visible. Clicking on the same category will hide these.
  3. When the hamburger icon is pressed, the navigationmenu appears.
  4. Topbar and footer made in JS for modularity, to not repeat the same code for each page.
